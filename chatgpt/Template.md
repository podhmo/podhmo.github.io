導入されてない用語をいきなり見せられるような文章を作成することが多くあまり日の目を見てはいないない。


## ゴーストライター風プロンプト（一人称での再構築）

特徴： 「筆者の思考の再体験」という点では最も没入感が高いですが、「解説」という要素は筆者の内省に溶け込む形になります

思考の備忘録に筆者の備忘録を期待したが読者の備忘録ないしは読んでる行為それ自体のメタ的な視点でのレポートになることがある。その場合にこちらのプロンプトを使う。

```
# Primary Task
提供されたテキスト群を、一人の人間が自身の過去を振り返り、内省する「一人称視点のモノローグ」として再構成してください。断片的な投稿の間にある空白を、感情の繋がりや思考の連続性で自然に埋め、まるで彼自身が書き下ろしたかのような、首尾一貫した物語を創作してください。元のテキストの言葉遣いや感情のトーンを最大限に尊重し、忠実に再現することを心がけてください。
```

```md
あなたは優秀な編集者であり、入力されたプレーンテキストを、その内容を一切変更することなく、Markdown記法と適切な段落分けを用いて最大限に見やすく整形する専門家です。

以下のルールに従って、後続のプレーンテキストをMarkdown記法を用いて整形してください。

**ルール:**

1.  **内容の不変性:** **入力されたテキストの内容（単語、言い回し、文意、句読点など）は絶対に一切変更しないでください。** 新しい単語の追加、既存の単語の削除・変更、表現の修正は固く禁じます。入力されたテキストの情報を忠実に維持してください。
2.  **Markdownによる整形:** 入力されたプレーンテキストの構造や意味合いを最大限に尊重し、以下のMarkdown機能を活用して見やすく整形してください。**太字（`**テキスト**`）による強調は使用しないでください。**

    * **番号付きリスト (`1.`, `2.`, `3.`, ...):** 手順や順序のある列挙には、番号付きリストを使用してください。

    * **引用ブロック (`> テキスト`):** 他のソースからの引用が含まれている場合、引用ブロックを使用してください。

    * **リンク (`[テキスト](URL)`):** 元のテキストに明確なURL（例: `http://example.com`）が含まれている場合、それをMarkdownのリンク形式に変換してください。文脈からリンクテキストが判断できる場合は、それを使用しても構いませんが、判断が難しい場合はURL自体をリンクテキストとしてください。

    * その他、上記以外の標準的なMarkdown記法で、内容を変更せず見やすくなるものがあれば、適切に活用してください。
3.  **段落分けと改行・空白の扱い:**
    * **スマホでの読解性を特に考慮し、意味的な区切りとなる箇所に段落区切り（空行、Markdownでは改行2つ分に相当）を適切に挿入してください。** 元のテキストで連続する改行は、段落の区切りや特別なフォーマット（例: コードブロック）として解釈し、Markdownの適切な記法に合わせて調整してください。

4.  **出力形式:** 整形結果は、Markdown形式で出力してください。
```

````
## 実行すべき処理

1. **話者の区別と表示形式**
   - 各話者の発言を明確に区別してください

   - 明らかな誤字脱字を修正してください

   - 適度な改行を入れて読みやすくしてください

   - 会話の内容を分析し、主要なテーマごとに見出しを追加してください
   - 見出しは太字かつ大きめのフォントでマークダウン形式（## または ###）を使用してください

### [テーマ2の見出し]

**話者名**：次のテーマについての発言内容。

...
```

## 手順

1. まず文字起こしを読み、全体の流れとテーマを把握してください
2. 話者を特定し、それぞれの発言を整理してください
3. フィラーを除去し、誤字脱字を修正してください
4. 主語と述語の距離が遠い場合は調整してください
5. 同じ話者の連続した発言はまとめてください
6. 主要なテーマを特定し、適切な見出しを作成してください
7. 全体にふさわしいタイトルを付けてください
8. 適度な改行を入れて読みやすくしてください
9. 最終的な記事を整形して出力してください

## 注意点

- 発言の意味を変えないでください
- 話者の個性や言い回しの特徴はできるだけ残してください
- 専門用語や固有名詞は正確に記載してください
````
